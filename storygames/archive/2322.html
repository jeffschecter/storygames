I want a new Web Toy for Making Relationship Maps

Dave Younce
February 2007 edited February 2007 in Play Advice
After reading this morning's Sin Aesthetics, entitled 'Relationship Web Builder Tool', I've been playing with relmaps all day. Everyone says the authoritative work on the stuff comes from 'Sorcery and Sex,' which I've never read. But it doesn't seem like there's any particular difficulty that would prevent the creation of a new Web Thingy that'll autogen relationship maps. I know about stuff like Mindmapper, and Gliffy, and that's not what I'm talking about - that requires _me_ to make the map, and I don't want that. I want a toy that makes maps for me.

Y'all know I'm all about the random generation, and this thought is a continuation of that. I want a webapp that'll take some parameters from me and spit out a pretty picture of a whole map.

I'm thinking it'll be a php page that'll have some controls - sliders or pull-downs - for the following scales:
# people on the map: 3<--->15
avg # relationships per person: 2<--->4
Sexuality: few sexual relationships<---->many sexual relationships (maybe another sub-scale for how rare/common homosexual relationships are in the map)
Violence: few violent relationships<--->many violent relationships
Business: few monetary relationships<--->many monetary relationships
Family: few family relationships<--->many family relationships

Then you hit 'submit', and it generates the map parameters as what we CS nerds call a 'digraph' in graphviz format and passes them to a new page running the WebDot stuff for graphviz, where your map is displayed for your viewing/printing pleasure. Maybe click on the image to regenerate a new map with the same parameters. Maybe down the line some kind of crazy Abulafia integration that lets it pull random names, personalities, etc from the random generators there.

So, what I'm looking for here is a) an example of somebody already having made such a tool, which will make me happy , b) suggestions, encouragement, or potential problems you foresee with such an application.

    I admit that this is a somewhat tardy response to your question about relationship mapping. I read your question with interest, and thought that you might find genogram software to be useful. There's a growing market among psychiatrists and clinical psychologists for software that combines some elements of geneology software (i.e., the family tree graphic display programs popular with the geneology crowd) with symbolic links that summarize major interrelationship issues, emotional connections, relationship histories, as well as genetic, social, economic, and health-related facts about particular individuals. There are lots of competitors in this field, and the software ranges from good to barely useable. As a starting place, the Wikipedia article on genograms is pretty good. I've used GenoPro, which is pretty flexible, but (unfortunately) has a pretty steep cost and limited shareware license. But there are certainly other programs out there as well.

    Posted By: Dave YounceI agree that would be more useful. The ideal system (for me, personally) would be able to do both directed and non-directed edges.
    Graphviz does this nicely. First, use the dot tool for two reasons: you want directed edges sometimes and because the layout is generally more legible than any other of the tools especially as the node density increases. Then use subgraphs to set edge types:


    subgraph { // show married people with arrows on both edge ends
    edge [label="married", dir=both];
    larry -> melissa;
    mom -> dad;
    }

    subgraph { // no arrows
    edge [label="bonded", dir=none];
    gordon -> fox;
    }


    I use dir=both a bit in my Reign rel map.

    More info:

    To use it, all you have to do is take the swf and put it in the same directory as an xml source file.

    If you call the xml anything but rmapper.xml (for example if you want to have more than one on the same page), you need to pass it in in a flashvar called xmlSrc. Make sure you put it in both the OBJECT and EMBED in the hosting HTML page.

    Example: 
    <object classid="clsid:d27cdb6e-ae6d-11cf-96b8-444553540000" codebase="http://fpdownload.macromedia.com/pub/shockwave/cabs/flash/swflash.cab#version=8,0,0,0"; width="800" height="800" id="rmapper" align="middle">
<param name=FlashVars value="xmlSrc=yourFile.xml">
<param name="allowScriptAccess" value="sameDomain" />
<param name="movie" value="rmapper.swf" />
<param name="quality" value="high" />
<param name="bgcolor" value="#ffffff" />

<embed src="rmapper.swf" flashvars="xmlSrc=yourFile.xml" quality="high" bgcolor="#ffffff" width="800" height="800" name="rmapper" align="middle" allowScriptAccess="sameDomain" type="application/x-shockwave-flash" pluginspage="http://www.macromedia.com/go/getflashplayer"; />
</object>

    If I may be permitted to perform a little thread necromancy, I've been experimenting with randomly generating relationship maps over the weekend. I've been wanting to perform some thought experiments with them, but creating them from scratch by hand was causing me too much anxiety, so I wrote something to create them for me.

    I've only partially completed the program, which ISN'T web based, instead being a standalone GUI application written in Python (so, in theory, it should be multi-platform; I'll test it on my Linux box before I release it). Well, it will be once I'm finished it.

    At the moment, I can randomly generate a 15 person relationship map. The names of the people are all generated randomly (I have two text files with 1000 given names each - one for male and another female. A separate surname file has nearly 89,000 surnames in it). The relationships are either sexual, violence, business or family. Each relationship is drawn in a different colour: sexual is red, violence is black, business is blue and family is green. Public relationships are rendered as a solid line. Concealed relationships - ones that I'm assuming are not public knowledge - are rendered as a dashed line. Each relationship is assigned an intensity rating, as a percentage. I'm assuming the higher the percentage, the stronger the feelings involved in that relationship are.

    I have made no attempt to assign any sort of meaning to any of the relationships. I guess that's going to be up to the person using the map to come up with some labels as to what the links mean. Still, I find that seeing a concealed one way sexual relationship as an infatuation is a good start. A bidirectional violence link might be an outright feud between the parties, and so on.

    The program is still very much a work in progress. I had originally wanted to have the Python program be able to drive Graphviz directly, but the library I wanted to use doesn't work with Python 2.5 (despite the claims on the website). So instead, the program spits out a dot file, which can be run through the graphviz programs manually to generate the picture version. I still need to write the GUI (using the WxPython library) to give the user a LOT more control over the input parameters; the ones there at the moment are pretty much hard coded, and so they give some weird results at times). I also need to add some more intelligence into certain relationship types. For example, the program currently doesn't worry about the sexes of the individuals when generating sexual relationships, so you're just as likely to get homosexual relationships as you are heterosexual ones.

    I've rendered on test map as SVG and I've uploaded it: Test random relationship map. The output of the graphviz rendered is still a little tangled right now. I've only been playing with it for a couple of days, and I've still got to figure out all the options to make it look prettier.

    Anyway, it's a start and it's been fun to play with over the weekend, when I've had a bit of spare time to sit and code.

    If people are interested, I'll keep on posting updates as I get more of a chance to work on it further.

    And if anyone is game enough to use the test map in a game, I'd be interested in hearing how it works
