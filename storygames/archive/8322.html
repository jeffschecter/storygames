    \n Yeah: generally slow all around. I share a grid container with many users: For the last few weeks I've seen continuous slowness, coming to a head today. I suspect that other users on the same grid are chewing up memory. After talking with my provider for a bit, I've upgraded mySQL to a stand-alone mySQL container of a constant RAM size, so that will eliminate Bad Neighbor effects. It'll cost me extra, but I'll have more of a leg to stand on when demanding better performance in the future.

    Note that I too saw the out of memory errors. First time I saw them on this provider, but I did see them a lot before I left Dreamhost: It literally meant that the sql server was simply out of memory, usually because some other doofus was consuming it all, and they didn't have a good method of determining who (note: It's not going to be a feature of bookmarks, etc: It's just the aggregate memory on the server as a whole, which I am sharing with probably hundreds, if not thousands, of other users). My new providers can pinpoint it better, but I figure that I'd go the extra mile and upgrade my service, so that if I need to yell I'll be heard that much more clearly.

    Ah, that's also the reason for my Upgrade Notice, above. the upgrade and move should be happening in the next day or two. When I move, the new server will allow me much deeper access into troubleshooting and analyzing mySQL behaviors, so if there are any lingering effects I'll be able to cure them from there.
