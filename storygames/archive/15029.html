    \n The only way to really produce a professionally made eBook is to do it by hand{*}. That is: know the standard and how it's implemented and learn how the code works.

    Calibre can perform conversions but it's an automatic tool, it only goes so far (and I know even some publishers use it... still, it's not a professional way to work).

    For example, ePub is really a .zip file, renamed, containing a standard structure of directory, a couple of xml files describing the contents and metadata, and .xhtml and .css files to hold the contents (plus images in jpg, png or svg format, fonts and so on).

    I've used Sigil (an "epub editor") to create the basic structure and then edited all by hand a couple test ePubs. It's not hard if you know html and css. The difference in the output is comparable to a hand coded website vs code auto generated by dreamweaver.

    Mobi (the format used by amazon) isn't much different, just extremely limited in what tags and features you can use (almost none, basically).

    {*} Obviously, once you know how to do it the hard way, you can code yourself a production pipeline to automate most of the process.
