I'm starting a new thread here for the techo aspects of Online Tabletop that only people interested in helping out with the development (however minimally) will care about. This includes people who will comment on technical questions even if they don't want to do any coding.

The issues of whether to do this and what other products are already out there have already been covered in the other thread.

First question: DHTML libraries. According to this Sitepoint article, the Big Four are:
  • Prototype (as used in Ruby on Rails, and upon which Rico and script.aculo.us are built)
  • the Yahoo YUI
  • Dojo
  • MochiKit
Does anyone have any experience with any of these or any recommendations to make?

So far what I know I'm looking for includes:
  • easy to create self-extending forms (which add new controls as the prior selections make them relevant)
  • rearrangeable/resizable/minimizable panels for the various parts of the interface (log, chat, tabletop, character sheets, die-rolling area etc.)
  • tabs
  • good support for AJAX/JSON (so that Shreyas can play Agon ;-} )
  • rich text entry controls producing valid HTML; preferably spellchecking addable
  • animations (appear, fade etc.)
  • drag-and-drop
  • easy to define a hotzone on a background image
  • nice quick image handling
I realize that some of these won't be in any of the libraries (there seems a remarkable lack of rich text controls in all of them, and I'll probably have to use something else like FCKEditor).

    Hi Mike, libraries wise, I'd pick one that matches your own coding style, and has a small footprint. I've noodled around with all of them in the past, I seem to remember prototype's documentation being a bit lacking, though this may have improved. JQuery is also worth a look, especially if you're familiar with CSS, they're in process of a slight rewrite at the moment though.

    For a chat app, it might also be worth looking at Comet (buzzword alert), where you use one long lived HTTP connection to act as a message stream. In apps with repeated polling it can have advantages (all those poll headers build up bandwidth over time). But has its disadvantages too (you tie up one of your requests, proxies can have issues).

    Also, if you're planning graphic drawing tools, IECanvas is an interesting option. Decidedly non-standard (it's WHATWG's canvas element with a wrapper for IE compatibility), but useful. Some of the libraries have drawing functionality too.

    For wysiwyg editors, TinyMCE is my current fave. It has limited Safari/Opera support.
