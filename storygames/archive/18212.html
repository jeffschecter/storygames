\n As has been illustrated by Stephen Fry, any randomly shuffled deck of fifty-two cards is statistically certain to have be unique from any other such deck. The actual probability of producing identical decks is 52-factorial; aka, 52!; aka 52 Shriek. (Approximately, 8.0658175*10^67.)

However, the most popular card games don't really make much use of the uniqueness of each deck: the effect is diluted by the combining of cards into hands, the division of games into rounds, etc. But most profoundly, it is diluted by our brains' abilities to see patterns. So often we see the pair, the full house, and even the royal flush is hardly uncommon. So might creating a game that maximizes the significance of a deck's uniqueness be a method of anti-story-gaming?

    \n You seem to be mistaking uniqueness in the sense of "not being the same" with uniqueness in the sense of "not being similar to any other decks". If you want to use the whole information content of the deck at once, then you should use the whole deck at once. The number of permutations of 52 cards tells us exactly nothing about the similarities (under any particular similarity function) of those permutations. One way you could do it is make a machine that generates story-games that's programmed by the permutation number of a given shuffled deck. If all numbers are valid programs, and all programs produce the rules for a story game that are different to the results of any other program, then you can use All The Uniqueness. To make sure that there are no similarities, you should make sure that each permutation N maps to the Nth incompressible string such that no previous game could generate that game and then run it. Fortunately for you, all you have to do to check that a string is incompressible is run all the previous strings and make sure that they don't eventually print out your string. After 10^67ish infinities, you should be all done. Although if you used a non-turing-complete programming language for the story-game-making machine it might be possible!
