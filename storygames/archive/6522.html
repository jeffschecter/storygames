\n I've just about finished a html and pdf final version of a game I've been working on for a while. I want to release the game for free, and I'd like some advice on how to do that. Ideally, I'd like a single site from which people could access pdf and html versions of my game (and any future games), and where I could possibly also post AP information, rules updates, reviews and so on. Pretty basic stuff. I also want to do it on the cheap. I'm somewhat savvy with html and css, but I'm no expert.

Here's what I've considered:

Building from the ground up:
This gives me the most control over the eventual look of the site, and means I can host the files exactly how I like. However, it seems very costly. Does registering a domain name really cost $70? Is there a cheap way to do this? I've tried free hosting before and found it pretty sub-standard. Another drawback is my dodgy html skills mean the site might look a little cheap.

Wordpress Blog:
This is free, and really easy to use. The main drawback is that I can only use one stylesheet for the whole blog, which means I can't use any fancy formatting for the layout of the html versions of the games (like callouts, sidebars, and so on). It will let me create multiple pages, and upload pdfs though, which is pretty cool.

Something Fancy With Google Docs:
I'm still researching this. It seems like there's a lot of potential I haven't discovered, and some combination of this and a blog or something might do what I want.

So, Story Games, what's the best way to do this? People who already have pages like this, how did you do it?

    \n I have the same philosophy about Anima Prime, which is why I've set it up on a drupal-based website. It's easy: get a host service somewhere, upload drupal into a directory, create a database via your host, then install. You can very easily implement the whole game on different pages, allowing visitors to comment on each page (and while it's called a "page," I suggest doing it section-by-section). It also includes an integrated interface for forums, blogs, polls, etc. that you can enable or disable and configure at will.
