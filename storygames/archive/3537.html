\n Looking for a mechanic that:\n
    \n
  • uses multiple dice: some range between 1 and 8 dice for each player "action" [a subrange within this can work)
    • \n
  • a player rolling more dice is better off, and some results can be more good than others (so degrees of success)
    • \n
  • a player does not feel obligated to maximize their amount of dice in order to get an acceptable result [I think this translates to: the number of dice will not effect the degree to which you can effect things, or at least only does so to some extent]
    • \n
  • low handling time [subjective, though]
    • \n
  • uses d6s [negotiable]
    • \n
  • given the choice to roll 1 die thrice vs 3 dice a single time, you more or less are better off rolling the three at once [very optional, negotiable, probably impossible and contradictory, this is the bonus round]
    • \n
\nThe mechanic that seems to work for me is: roll dice, take the highest single die, and compare to (TN / task ladder / badger / other person's highest die). A player rolling just one die can still get the maximal score (6) but there is still pressure for more dice. Though maybe not enough. Not sure.

    \n Okay, you could use the 1000 Kings dice. Here's how they work.

    Each side has a pool of d6s. 3-20, let's say. Accumulate them from various places. Don't have less than three.

    Each side rolls all their dice. Take out all the ones and set them aside.

    Highest die wins.

    If there's a tie for highest, reroll all dice which are tied for highest in a new roll. Set aside the ones. Highest die wins. Repeat until there is a winner.

    Sum total of all 1s = degree of success.

    One way to stretch the diminishing returns of "some d6s, pick the highest" is to pick some other thing to watch out for too, like maybe "for each pair you have, add 1 to the result" (more stretchy) or "for each (max) you have, add 1 to the result" (less stretchy).

    \n

    You can add some degree thing in there by comparing the winner's result to a loser or standard.

    \n

    I'm not sure about the math behind this, but if my guessing's right, it keeps the 'diminishing returns' effect of more dice, but it flattens out the curve to some extent, so you can get into higher numbers without hitting a stark effectiveness plateau, and it adds a longish tail in the higher results, which I always think is nice.

    \n Building on Daniel's thoughts, what about:
    \n
    \nRoll as many dice as you like within whatever range you set (1-6).
    \n
    \nSet highest die(ce) aside -- they count for winning.
    \n
    \nAll other dice you rolled count against you in some fashion (say, exhaustion if physcal. Alternatively, they could count against a "timer" which ends the conflict when it reaches a preset value).
    \n
    \nMargin of Success = least number of dice set aside by either side (so, if you set aside 2 and won, and your opponent set aside 4 but lost, then you win with MoS 2).
