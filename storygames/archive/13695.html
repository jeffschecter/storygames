    \n According to YSlow, http://story-games.org/forums/comments.php is slow because of the following:
    * No Content Delivery Network (CDN) but 39 static components.
    * No far-future Expires headers on those 39 components, so they re-DL every query.
    * JavaScript not at page bottoms, thereby blocking parallel downloads.
    * 38 components with misconfigured ETags (gotta admit that I have no fucking clue what that means).
    * Lots of unnecessary HTTP requests:
    --This page has 9 external Javascript scripts. Try combining them into one.
    --This page has 5 external stylesheets. Try combining them into one.

    99% of this "bad" structure is caused by the template for the site that's in use and how it's structured (JS and CSS pages). With some judicious hacking, one could combine the CS/JS stuff pretty easily (and with a bit more dangerous hacking, move many of the JS calls to page bottoms). I'm not sure what to do about Expires headers on graphics, or whatever the hell ETags are; and setting up a CDN is something more on the provider's shoulders than the site owner's (though the latter might be asked for more money a month for the former to employ a CDN).

    \n One could set up expire headers on graphic components without direct access to the server with PHP script, I think - basically set all graphic referrals to point to a script that creates the appropriate header and then sends the header and the graphic. I haven't done it myself, either, but that's what I'd try first. Goes into web programming territory pretty heavily, of course. I don't know what ETags are, either.

    Notably some of David's slowdown points are not server load issues but rather issues of end-user experience: moving script calls to page bottoms does not reduce the server load at all (it's still the same amount of data that has to be loaded), but it does ensure that a proper web browser can display a page faster, as it doesn't need to wade through the Javascript code before starting to display the page itself. The point about content expiry is a good one, though; it's clear savings for the server if the admin manages to set it up so that all the forum regulars that load dozens of pages a day don't reload all the static components each time as well.

    \n That's a good point about merging the files, that would certainly help with the server load. I suspect, though, that it would be less intrusive on the Vanilla structure to figure out functional expiry rules for the server so that the individual browsers would stop reloading the javascript altogether. I haven't looked into the guts of Vanilla, but if it's anything like other modern web programming projects, it's likely built so modular that it's a big customization hassle to reduce the number of individual files.

    \n This Greasemonkey script is bloody horrible, but works:

    // ==UserScript==
    // @name Ingore Story Games Threads
    // @namespace wordman
    // @description Ignore threads on the Story Games forums
    // @include http://story-games.com/forums/
    // @require http://ajax.googleapis.com/ajax/libs/jquery/1.3.2/jquery.js
    // ==/UserScript==

    function main()
    {
    \t$('ol#Discussions li.DiscussionTopic a').each(function(i, link) {
    \t\tvar $this = $(this);
    \t\tvar topic = $this.text();
    \t\tvar hide = false;

    \t\t// Some choices here:\t\t
    \t\t// To match exactly...
    \t\tif (topic == 'Post-Snow White Horror Adventure') {
    \t\t\thide = true;
    \t\t}

    \t\t// To match partially
    \t\tif (topic.search("Diplomacy") != -1)
    \t\t{
    \t\t\thide = true;
    \t\t}

    \t\t// To match a regular expression (this one
    \t\t// matches any topic containing four numbers in a
    \t\t// row, such as a year)
    \t\tif (topic.search("\\\\d\\\\d\\\\d\\\\d") != -1)
    \t\t{
    \t\t\thide = true;
    \t\t}

    \t\t// You can keep adding new items like this.

    \t\t// Then... hide if we saw something we didn't like
    \t\tif (hide)
    \t\t{
    \t\t\tvar discussion = link.parentNode.parentNode.parentNode;
    \t\t\tdiscussion.style.display = 'none';
    \t\t}
    \t});
    }

    main();
