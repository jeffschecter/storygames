\n We're seeing a lot of "500 Internal Server Errors" every weekday AM (prime hours), and it sucks. Sorry for the inconvenience. My host is awesome in all respects save anything mySql-related. Again I trek to find new web storage. What I'm planning on doing is this:

1) Hunt for a new host. Will attempt to find something pretty high-end, respectable, and hopefully VMWare/virtual. Primarily, I want a vendor that can assist with DB problems, rather than just say "Looks like DB problems".

2) Move to new host, if a good one is found.

3) Then monitor. Vanilla, as forum software, is pretty new and for all its awesome bits it still has a lot of untweaked parts (especially in regards to the cool extensions). If it simply weighs too much when moved to a new server, I'm going to move to (garg) vBulletin or the like, albeit a totally tricked-out version that feels more like what we have here.

I'll keep folks informed as things develop. It may be the case that my host notices an issue and kicks something, returning normal service levels. We'll see.

In the meantime, if you write long posts, please Ctrl-A to select all, Ctrl-C to copy to your clipboard, then hit "Submit Post". If you get a 500 error, at least you have the text saved. In most cases, if you get the 500 server error, hit Reload and select "Post Data" it will work, but just in case keep a backup of longer posts so they are not eaten.

Thanks!

-Andy
