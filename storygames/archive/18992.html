    \n Are you asking about the initial setup when people are buying stuff, or once play has started?

    When buying things at character creation it's simply a case of "if a player calls in a favor and that NPC is on the map already, draw a line and invent a relationship to something already there. Otherwise put the NPC on the map but don't attach them."

    If a PC is talking to a connection about "stuff", the GM rolls a die, consults that NPCs connection table, and puts the result on the map (and draws a line to something that's already there). If the thing you rolled up is already on the map, link it to the NPC you're talking to.

    If the PC is instead asking the connection about something specific (instead of just hunting for general information), the only difference is that you don't need to roll and can instead pick a node off the list that makes the most sense. Everything else applies so if you pick something that isn't on the plot map you write it down and connect it to something that's already there, and if it is on the map already you connect it to the NPC whose list you just consulted.

    Mechanically you get identical results, just in the case where you're asking about specific information you skip the roll and instead get to pick something.

    Cheers!

    \n Unfortunately, there are really significant errors in that summary. In fact, I would go so far as to say that it's completely wrong in every important way.

    There are four circumstances in which you will add information to the plot map:

    1) Creating the mission seed
    2) When a connection is hit up for a favor
    3) When a connection is leaned on for a lead
    4) When a connection is inquiring about a specific node

    Mission Seed: Randomly determine three nodes using the master table from the transmission.

    Favors: The first time a connection is asked for a favor, you add the connection to your plot map. The second time the connection is asked for a favor, you connect them to the plot map by selecting a node of your choice. (This happens regardless of whether the favors are being requested during character generation or during runtime.)

    Leads: If the characters go to a connection to ask for help in a generic way (just fishing for a lead), you roll on that connection's table. If the result isn't on the plot map yet, you add it to the map and connect it to a node of your choice. If the result is already on the plot map, you add it to the map and connect it to the connection being questioned (the connection is involved somehow).

    Inquiring: If the characters ask a connection about a specific node that's already on the plot map, you follow the EXACT SAME PROCEDURE as you did for leads. The only difference is that you connect the node you generate directly to the node the characters are asking about.

    In all circumstances, of course, the GM can simply choose to ignore the random roll and choose the node which makes sense to them.

    It should be noted that the rulebook doesn't address one specific scenario: The PCs are inquiring about a specific node and the random roll gives the result of the same node they're asking about. I would recommend, following the general logic of the system, that if this is the result the node they're asking about should be connected to the connection they're talking to (which means they either asked exactly the right guy about it or exactly the wrong guy about it).

    The plot map may also be modified during maintenance between sessions, but that's a separate kettle of fish and is mostly just a "use your best discretion" kind of thing.
