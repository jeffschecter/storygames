\n As a programmer with as much of an interest in the software around games as in the games themselves, I would like to learn more about the story game community's relationship with gaming software: how you use it and how you would like to use it.

I'm interested in your thoughts in general, but here are a few points that you might consider:\n
  • For which parts of your games do you use software?
  • For which parts of your games would you use software if you could find it?
  • When specialized software (such as a character generator for a specific system) is available, do you feel that general-purpose software (such as a digital spreadsheet) is still better?
  • On which devices do you prefer to use software?
  • Do you find that software is better or more useful in rules-heavy systems, or the other way around?
\nThis isn't just about game-specific software (such as character generators) but about software for any aspect of the RPG experience, including things like mood music, note-keeping, and session scheduling.

Note that I don't want to start a discussion on whether software and computers are appropriate at a gaming table; this is just about how you as individual users feel about the software that you do use or would like to use.

    \n I'm also a programmer and design lots of online tools for my own prep. Random generators are a big part of that work. And pixlr.com for graphics. I still feel weird about devices at the gaming table, however. That includes my own. I do prep on my PC and print it out for actual play.

    I'm sure the feeling of devices at the table being weird will change as time goes by. One day they'll be inside our heads. However for now, I still think devices in the players' hands are an impediment to immersion and can often work against the flow.

    So in summation: WEB-BASED GM PREP TOOLS.

    \n What Alsf said.

    For my personal experiences:

    * For Burning Wheel character creation, web tools speed up a lot of the process. I familiarized myself with the lifepaths by stating up a bunch of NPCs that way when I was learning the system.

    * Pinterest for reference images.

    * I've used music/sound loops to set the mood. Seems to work best with long background soundscapes rather than trying to do in-the-moment sound effects. Tabletop Audio is probably the gold standard for this, though I've tragically not gotten to make as much use of it as I'd like to. I've also got the DMDJ app but have yet to use it in anger.

    * At the actual table, the two big problems is that when you're looking at a screen you're not looking at the other players, and that they tend to be really distracting. Even when players were using a device just for reference, it was tempting when there was too much downtime. I now basically only buy roleplaying books that I intend to run as physical copies, because it's hard to introduce a new game to people by passing around a tablet.

    * PDFs for reference/prep/secondary reference still get used; and smaller games work fine from PDF. But if I can't print out the relative information on a few pages, it's much harder going. Plus I can have multiple copies of reference sheets and things. I've got what amounts to a Dungeon World starter kit in a three-ring-binder, with character sheets, reference pages, and so on.

    * Having the players keep a record of the sessions on a campaign blog seems like a good way to keep them engaged beyond the session, though I haven't really had a chance to follow through on that.

    * I use Google Docs (or whatever they renamed it to this week) for a lot of planning stuff, especially text-heavy things. I've also used Obsidian Portal's wiki for campaigns.

    * I've run a one-on-one Google-docs Dungeon World play-by-post game. Could use an edit notification to let the other player know that they can respond.

    * Going further afield, there are several play-by-email systems. I recently wrapped up a campaign using Callisto, and wrote a data visualiser to show the relationships between the characters as they wrote letters to each other.

    \n I very seldom use anything other than internet for research, or simple word processors, readers, and graphics programs (and then print them out for the game). The exception is Campaign Cartographer and other Pro Fantasy software for map making. For some few games I will stream a music playlist for mood.

    All my work is done on a PC.

    All my work is done during the prep phase, other than music on the rare occasions it's used.

    I have very very rarely used something on my android phone...like a dice rolling app. This is usually only when an impromptu game springs up while away (like camping) and I wasn't otherwise prepared with gaming materials.

    Generally speaking our groups consider roleplaying a pen & paper, at the table endeavor, and so don't use much tech.

    \n Considering my current gaming is all play by forum, the software I use is:

    Web browser (mostly for the forum, google docs, and davesmapper)
    PDF Reader

    In the past, I have used quite a variety of software including programs written by myself to create/visualize star maps, generate price lists for magic items, and other prep/reference type stuff.

    One summer I ran an SF campaign with a Compaq "portable" PC (pre-laptop days) running Turbo Sidekick to provide a notepad for keeping session notes, running a program to do fuel calculations for jumps in the foreground.

    What I have NOT used a computer for is music, sound effects, video, or images during play. I find the effort of pulling up such stuff to be disruptive to play.

    \n "For which parts of your games do you use software?"

    Random tables, mapping, and NPC generation on-the-fly

    "For which parts of your games would you use software if you could find it?"

    preparation, and random stuff during the game.

    "When specialized software (such as a character generator for a specific system) is available, do you feel that general-purpose software (such as a digital spreadsheet) is still better?"

    Not really, unless someone has created a macro or script to handle the task. Specific tools are cool here. I do use Evernote for note-taking and prep planning, though. So that would be an exception.

    "On which devices do you prefer to use software?"

    Android, for sure. Especially Android phone during the game, but also tablet for prep.

    "Do you find that software is better or more useful in rules-heavy systems, or the other way around?"

    Can't say for sure, but it would seem to me that it would be more useful for rules-heavy games. That's just a feeling I'm having, at the moment.

    \n My group uses MediaWiki with the Semantic extensions to keep track of PCs (for simpler systems at least), NPCs, and session logs. I love the flexibility of the Wiki combined with the power of Semantic and templates - it's nice to be able to click on an NPCs name in a session log and see their basic information and a list of other sessions where they turned up, or to answer "who was that guy we talked to who knew about trains" by using the search feature. It's proven useful enough that nobody minds me sitting there taking session notes as we go.

    \n I think a pretty cool function/program would be something which allows you to plug in a variety of random tables (ideally even from pictures or PDFs), and access them on the fly during play.

    Whenever a table comes up, it automatically generates a result from it (using the appropriate randomizer).

    For instance, I could load mine with three "Medieval Village" random tables I found online, and one I wrote myself, and (before play) identify them with the appropriate die types. In play, I could just type "village" into a search box and the three tables would pop, each one with a number generated, so I can immediately look down and read the result.
