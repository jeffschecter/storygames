As mentioned in the Stuff to Watch thread, the folks over at Opera Software have released the latest version of their browser Opera 10. However, whats interesting about this release is that it includes a new featureset called Opera Unite that allows the user to run a webserver of sorts directly from their browser, without any additional software. Right now it supports:

\n
    \n
  • General file-shareing
    • \n
  • Photo galleries
    • \n
  • Music streaming
    • \n
  • Chatrooms
    • \n
  • Webpage serving
    • \n
  • and a 'Fridge" where you can leave notes for the server's host
    • \n
\n

All of the above is available as long as the host computer is on and connected to the net, with the Opera browser running. All of the services can be made either public, private, or protected with a unique password. This might be a useful tool in many respects, like say, hosting a gaming session over the web with all your relevant resources and materials (music, pictures, maps, chat, etc)

\n

I'm currently messing around with it myself (music and fileshare protected under the password: mythopoetic1). If anyone else is trying this out, drop a link to your Unite page in this thread.

    \n Renato - that's fully half of the story, actually: the Opera folks are running a proxy server that, to the best of my limited understanding, works like DynDNS. When you switch on the Unite functionality in your Opera browser, it will automatically inform the Opera.com proxy server of your presence online, and it fires up yourusername.unite.opera.com or something. So that's findability taken care of.

    As for ports, lots of other software that technically "acts as a server" has sorted it out. I'm sure Unite is no exception.

    The potential of this stuff is almost better thought of in terms of chat clients than in terms of web servers.
