    \n If you want programs to calculate dice probabilities, there are several possibilities:\n
      \n
    • UeberDice - Java-based, easy to set up and easy to view graphs of probabilities. Unfortunately, sometimes it will get stuck in an infinite loop with more complex probability calculations.
      • \n
    • DiceForge - Javascript, easy to set up and easy to view probability graphs with. Documentation is a little sparse (still kind of in development)
      • \n
    • Troll, mentioned above - more complex to set up than the previous two, but a more mature program, and its dice expression compiler is probably more correct
      • \n
    • Excel (or some other spreadsheet) - the easiest way is probably to use its rand() and randbetween() functions and whatever conditionals you need to create the form of the roll, then duplicate that formula over hundreds of cells and count the numbers of each result. (Monte Carlo method)
