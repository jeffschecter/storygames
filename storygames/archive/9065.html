    \nNot natively, no, lame huh? I've been using sscanf... we'll see how far that takes me

    Ok, in terms of the roller, here's my latest idea: I'll build in support for any die operation that isn't totally system specific, like your rank() function, like the (d4,d5) + (2d6,0) thing, or anything else people suggest. Then, I'll add the ability to define shortcuts for fancy rolls (using simple string substitution). I'm not familiar with A Wicked Age, but you could go to the shortcut-building page and make a shortcut named "wa" which would look like this (or something similar) - rank(d8, d6, d4) + (<1>d6,0,0), then you could type "wa 2" into the field and it would make your roll. Make sense? If you typed "wa 3" it would roll 3d6 instead of 2d6. It would do a simple string substitution with the numbered parameters, and the final roll would be separated by spaces, so if you had two parameters you might type "wa 3 2", or even "wa 3 d4".

    I'll also add some sort of table functionality where you create custom dice. So you might go to that view and create one called "F" then enter "+" "-" and " " Then you could do something like 4F or 4dF and it would 4 of them.

    What do you think? Does that sound flexible enough for everybody? That way you could create rolls easily without it being a big pain in the neck or requiring difficult UI implementations (remember I still suck at iphone development). All we would need after that is a server to host everybody's public shortcuts and tables and make them shareable. ... Maybe one release at a time
