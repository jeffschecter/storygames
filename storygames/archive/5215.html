    \n There are externally-focused Keys (Conscience, Impostor) and internally-focused Keys (Orphan, Love). As a GM, you don't need to worry about hitting Key of the Orphan so much, but you better provide a weakling in distress at some point, because it's hard for the player to work that up himself. So looking at your player's Keys allows you to pare down your workload a little by focusing on hitting stuff they can't easily hit themselves. In a perfect world you persuade different players to take opposed/concordant externally-focused Keys (Conscience + Bloodlust), then sit back and let the game run itself.

    \n I love having conversations with myself.

    I'm going to add a technique to this thread. I have not yet attempted this in a real game, but I'm pretty sure it'll work:

    When setting up the game, I will first create a small stable of NPCs. (Hopefully, with as much player input as possible--ideally, we would all make these together.)

    Next, when the players create their characters, I will ask them to draw a connection to an NPC (they can create a new one, if none of the existing ones work) for each Key they take. Explain why. You may have to make up some past event.

    So, if you have the Key of Unrequited Love, it's going to be for one of those NPCs--choose one. If your second Key is Key of the Daredevil, maybe you can tell the group how you won or lost a crazy wager against a second NPC.

    Finally, I'll add connections (mostly antagonistic/conflicting ones) between NPCs that aren't tied together already. (My general rule is that each one needs two relationships before I stop drawing connections.)

    The result should be a nice relationship map, full of suggested scenes, conflicts, and other fun.
