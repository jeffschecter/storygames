\n So I'm translating this game from Japanese to English. I do most of my work at home, but some work I can squeeze in during small periods of peace at my hectic day job. I basically keep two copies of the docs on each PC, and mail myself when I update one of the files.

Fuck that. I finally got with the program, and uploaded my active docs to Google Documents. Now I can work on one standard doc at home or at work, saving along the way. Also, the thing that I assumed that it couldn't do - multi-language - it can do, and easily at that. I can switch up English and Japanese in a heartbeat.

The only problem: I often want to spellcheck a word Right Then and There. However, when I click Spellcheck, it checks the entire doc at once. I wish there was the simple "right click on word to spellcheck it" feature. I'll email google about that.

Anyway, just wanted to put that recommendation out there. Anyone else have experience with google docs, positive or negative?

    Besides, speaking of collaboration tools, you might take a look at the so called Source Control Management systems, that for the non-techie among you are systems that allow multiple people to edit the same corpus of (text) documents tracking changes and (more importantly) allowing different persons to work on separate parts of the same file without problems, and also to resolve editing conflicts.

    They are most commonly used for the source code of software projects, but anything textual will work just fine and dandy (txt, html, rst, tex, and so on).

    For an introduction, check out the second chapter of the Svn RedBook: SVN Book Chapter 2.

    Another nice fact about Subversion (and other similar systems) is that you work on a local working copy, and need network connectivity only to commit your changes. Even viewing the differences between the repository copy and the working copy can be done offline (there is a "hidden" reference copy).

    \n Or, to balance things, use a somewhat richer format that is still text:

    \nWhat's so difficult about emailing back and forth between two computers? I do it all the time between my work computer and my home computer, plus my laptop on occasion, not for personal projects but for work stuff that will get me fired if I screw it up, and I rarely have any trouble with versioning.

    So here's my simple non-software solution:

    - When revising an article, I get obsessive about using "save as" to leave the prior version intact and create a new version with the same name and a different version tag: v1a, v1b, v1b, and so on -- I've gotten as far as v3i at leas once.
    - When adding to an ongoing notes file, I email the file to myself at the end of every day. If I actually make modifications to the file at the non-default location -- i.e. I do an interview from home -- I use "save as" and retitle the file "[whatever the name is] WORKING" (notice the big block caps). Then I email it back to myself and, first thing on arriving at the other location (i.e. work), I save the "WORKING" version over the version there.

    - When I don't use either of the methods above, or sometimes in addition to them, I dump backup copies of individual files an entire project folders into a catch-all folder labeled "Holding." That way, if I really screw up, I always have a relatively recent version in the Holding folder.

    Plus I save all my work-related writing in .txt format for simplicity's sake, but then I have a paid, professional layout crew of five people to handle the aesthetics for me there.

    Basically I have zero confidence in any method that requires constant, rather than intermittent, Internet access to function, and I have very little confidence in any file format that requires a specific program to open it.

Personally I'd prefer the system I'm used to from work, where one person, the editor, acts as a gatekeeper for all substantive (non-typo/spelling) changes, such that anyone else has to send in a proposed change and then the editor/gatekeeper has to make a conscious decision whether to incorporate it in the working document or not.
