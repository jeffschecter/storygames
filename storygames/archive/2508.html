    \n Naw, it's my host. In the past few months I was moved from a "slow mysql server" to a "fast mysql server, apparently co-hosted with folks who fill up their databases and explode shit, causing the server to be rebooted at least once a week". I'm seeing if they can buy me something shiny for my troubles.

    Just some info so you know what's going on next time:

    "Fatal, Non-Recoverable Error" == Generic error with DB, server, etc.

    "Gateway timeout" or the like: Mysql server is down or problematic. Usually during a reboot. Expect the system to be back in a few minutes.

    "Error 25" - Can't connect to database. Usually means that something exploded on another account and they're rebooting the mysql server to clear it. Usually goes away after 5-60 minutes.

    "Error 28" - This is a new one. Some other account on my mysql server fills up the DB, takes every last available byte. Knocks out the rest of us. Usually this goes away after an hour or two, because first the provider has to be made aware that there's an issue, then do something about it.

    Hope that helps.

    Anytime I notice an issue that I think will be around for more than 10 minutes, I tend to edit the front page of Story Games, www.story-games.com to give a status update.
