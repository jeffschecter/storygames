    \n Most die pools are ways of getting a set of integers from a die with a predefined frequency. For instance, fudge dice get their result from the set [-1, 0, 1], implemented using a cube, so physically it's [2:-1, 2:0, 2:1].
    \n
    \nExalted dice are 1d10 read as [6:0, 3:1, 1:2], neglecting botches. If your program showed the sum of the results and then, separately, the results of the "dice", you could visually pick out botches, but that would be hard to syntactically express to a program.
    \n
    \ndExalted=(1d10=>[6:0,3:1,1:2])
    \n
    \nroll 5dExalted
    \n5dExalted=4 (3, 5, 7, 9, 10)
    \n
    \nI'm not sure what a "sorcerous skill" does to a die pool, but I know that BW dice are just a variable TN binary dice, so if grey dice succeed on a 5-6 on d6, that's [4:0,2:1].
    \n
    \nI'm sure someone else has tried to solve this problem in a generic way before, but I don't know what they've come up with.

    \n You could just extend JavaScript or something with some dice expression stuff, but here's a stab at a dice language.



    You want to be able to define a die as a set of faces, specify dice color, and specify certain operations on dice. You probably want to be able to write custom functions for new dice.

    Simple dice: d6, d8, d11, d%

    Face sets: d{1-6}, d{3,5,7,9,10}, d{-1,0,+1}

    Colors or pools: d6[red], d{-1,0,+1}[black], 9d6[SignalPool]

    Variables:
    @SignalPool = Pool(9d6)
    Strength = 3d6

    Operations (Excel-style functions):

    Count(dice pool) returns integer
    Sum(dice pool) returns integer
    CountIf(dice pool, condition) returns integer -- e.g., CountIf(6d10, =10)
    SumIf(dice pool, condition) returns integer
    Filter(dice pool, condition) returns dice pool
    Highest(dice pool, how many) returns dice pool
    Lowest(dice pool, how many) returns dice pool
    Combine(dice pool, dice pool) returns dice pool
    Remove(dice pool, dice pool) returns dice pool
    Roll(number, faces expression) returns dice pool
    Eval(dice expression) returns integer

    Old World of Darkness dice:
    function oWoD(number, difficulty) {
    @mypool = Roll(7, 10)
    successes = CountIf(@mypool, =difficulty)
    botches = CountIf(@mypool, =1)
    return successes - botches
    }

    > oWoD(5,7)
    2

    The most intensive grammar, actually a scripting language, is Torben Aegidius Mogensen's Troll: http://www.diku.dk/hjemmesider/ansatte/torbenm/Troll/ . It's comprehensive, and most helpfully, allows you to compute probabilities--it even does histograms! Unfortunately it requires a somewhat obscure compiler, Moscow ML, though it is available for most operating systems.

    \n

    Troll is the XML of dice-rolling: theoretically human-readable, but really you use it to write rolling programs.
