\n I was thinking about how could be a game system, the mechanics be kept fresh and exciting and interesting with the possibility that it's meaningful. And I realized that it's only possible if the rules are changing.

Which means that if a totally new situation arrises, there shall be a totally new set of rules for totally new mechanics.

Like when there was the first Star Trek episode which had Romulans in them. That was totally like a cold war submarine thriller drama, like Red October. So you had to dump the Star Trek rules (well, most of them, but keeping the baseline) and take the submarine rules instead.

If you as a GM want to give the players totally new experience in a scene, then you have to create totally new rules system for that scene. Every time. You can't just use the old rules every time. So the rules have to be intuitive or learnable or something like that, and the GM needs some kind of "meta-rules" which help to create rules.

I think that the tests and challanges of FATE are going in this way (as I can build very different tests and challenges), although the basic mechanism stays the same.

What do you think? Is it possible to create working and exciting new rules on a long term on a weekly or bi-weekly basis? Maybe the players learning curve could be used as the new rules' testing curve. Each adventure would be a new rules system which culminates in the final session when the players learned to use the new rules set quite well and the rules got refined by the players playing it. The question is how should it be done without getting disasterous results? How slow should the rules be developed to stay with them on the safe ground, that they won't have to get dumped, but can be refined for the first go?

    \nWhat do you think? Is it possible to create working and exciting new rules on a long term on a weekly or bi-weekly basis?

    Depends how many thousands of dollars you're willing to spend hiring full-time designers and playtesters...

    Which is to say, no. I don't think it's possible. It may be possible that once in a blue moon a designer out there has a flash of insight and designs a fully-working, interesting system in a single day or week... but mostly mechanics require significant playtesting, revision, etc. Design just isn't that easy.

    Now that doesn't mean you can't achieve a similar effect -- you do it by using existing mechanics, from other games, etc. But the idea of creating brand new, exciting mechanics every few weeks sounds like a pipe dream to me -- if you have a group that can do that, I hope you're publishing and designing games too.

    That is, you can do it by inventing the new rules set as you go along. Every session you start with a blank slate, rules wise, import some stuff from the last session and with the focus of this week's session in mind, add new rules and change them as you go along.

    --

    Also, I don't agree with Daniel, above. You can definitely come to the table with new, functional rules every time if you have a meta-system that guarantees them to be functional. This meta-system won't be that easy to design, but it's certainly possible. I suggest using Don't Rest Your Head as the base. You can do almost anything by having a bunch of dice pools that affect each other.
