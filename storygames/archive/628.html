Foundry MUSH -- chat and play with other indie folks

Adam_Dray
April 2006 edited April 2006 in Play Advice
I've put up a MUSH server for the purposes of hanging out online with cool indie gamer folks and maybe do some gaming, too. I've written some dice rolling code and a virtual deck of cards for people to use. It should be suitable for games that don't require complicated dice or cards. I'm amenable to coding systems for other games, if people want them. I'll probably install some other non-RPGs, too, like Boggle, just for fun.

Please, create a character there and hang out. It's a great place to discuss ideas and get to know people in real-time chat.

Connect to the Foundry using plain ol' telnet (ick) or via a mush client application like SimpleMU for Windows, Cantrip for Mac, and TinyFugue for UNIX / Linux (links to all these and more on that page).

Oh, if you're not familiar with MUSH, there's a decent beginner's guide and there's some basic help on the server, too.

Vaxalon and Blue were discussing playing With Great Power online. Chat them up in that thread if you want to join in.

The game's address is legendary.org 7777. That's port 7777 on legendary.org. Hope to see you soon!

    MUSH is a server for text-based virtual worlds. Foundry MUSH is a chat-and-play server for indie folks.

    It's like a cross between IRC and World of Warcraft. Like IRC in that it's text-based and you can chat and stuff. Like WOW in that you can move around (in a text world), look at things, and manipulate objects. It's more like ZORK than WOW, but few people seem to remember that game.

    Ahey, :)

    Because I'm crazy, I created a table to play Capes.

    The table keeps track of all of the gamespace, but it does not enforce the game rules. In other words, if you want to turn a die all the way up to 17, you can.

    What the table knows about:
    -Players, including story tokens and inspirations
    -Characters, including drives and stored debt
    -Conflicts, including sides, dice, claims, allies and staked debt

    What the table doesn't know about:
    -Character traits
    -Whose turn is it to act, start a page or frame a scene
    -Drive exemplars, or indeed, free conflicts of any sort
    -Actual game rules

    The table was created so that multiple copies can exist in multiple rooms. There is a test table in the main lounge with a sort of an example game situation.

    By the way, Adam, either the table parent needs to be chowned to some sort of wizard user or I need a truckload of pennies. The table creates the conflicts, characters and player seats as MUSH objects.

    Have fun and let me know if anything's broken so I can fix it.

    One of the more useful objects you can have in a MUSH environment that doesn't really apply to IRC is a descer.

    A "descer" is an object you carry around (or code you put on your self-object) that you can use to quickly switch between one description and another. Very useful for GM's who switch between NPC roles. This allows players who have forgotten how a particular NPC looks to quickly bring up that description without having to pause the game to ask the GM for that information. Some of the descers out there are very sophisticated.

    Rooms can also have similar code.

    • Adam_Dray

      My server legendary.org primarily hosts my RP MUSH FiranMUX, which has hundreds of active players, hundreds of thousands of objects, and an in-memory database that reaches about 800 MB. I've implemented quotes partly to prevent an impact to that game.

      I'm happy to host Foundry and let it grow as necessary. I'm happy to let people play around with building and coding on Foundry, since there is definitely a game-design tie-in. The budgets and quotas I've set up on Foundry are sufficient to learn how to be a good MUSH programmer. If you want to build a RP MUSH with dozens or hundreds of rooms and objects, though, Foundry isn't the place.

      If you want to build a giant thing, Foundry isn't the place for it. It's easy enough to download TinyMUX for Windows and host it on your own box (if you have Linux, it's very easy to install a TinyMUSH or TinyMUX server of your own and I can help with the installation). I can even suggest a place that will give you access to a free server of your own for development purposes (and charge you a small fee when it opens to other players).

      I just want to keep Foundry focused on its purpose: chatting real time about indie games and hosting real time indie games on a virtual tabletop with a bit of code to help.
