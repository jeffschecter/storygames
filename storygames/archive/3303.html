\nWhat's a TheoryGame?
\n
\nA theory game is a game built around a nugget of theory. It aims to experiment with and demonstrate to others the working of this particular chunkk of theory. Ideally, the act of reading and playing the game teaches, demonstrates and critiques the chosen aspect of theory, all at once.
\n
\nAlso ideally, TheoryGames are small (both in page count, wordcount, and in terms of time to play and other social footprint costs), and free in the monetary sense (like the theory is, mostly).
\n
\nWhy TheoryGames?
\n
\nTo demonstrate Confused on what IIEE is? Game X shows it in action!
\n
\nTo explore - Perhaps there's an aspect of theory that you think doesn't get much use, or might produce soemthign interesting if explictly applied. Find out!
\n
\nTo critique - By using the theory we discover its limitations and flaws.
\n
\n As a challenge - Not sure how a particular bit of theory is useful Just looking for some structure for you next project? Make a TheoryGame!
\n
\nAs an answer - "so what use is this theory, anyway?" Well, it directly influenced these games in a way which is really obvious!
\n
\nPossible Ideas
\n
\nMake a game that:
\n
\n* Demonstrates explicitly in game the difference between Intent, Initiation, Execution and Effect.
\n* Demonstrates the difference between conflict resolution and task resolution
\n* Demonstrates the four types of Authority (as Ron talked about here) and ways in which they might be divided up.
\n* Demonstrates exactly why the Czege Principle is right (or ISN'T right, for the daring).
\n* Demonstrates why deprotagonisation is always rubbish (or might not be, for the VERY daring! Try making a game in which the players are deliberately deprotagonised!)
\n* Demonstrates the differences between the five Elements of Exploration (Setting, Character, Situation, System, Color). Maybe each player can only provide one of each? Go nuts!
\n* Demonstrates the differences between Karma/Drama/Fortune resolution methoids, and their effect on game play. (perhaps the game uses Karma for some things, Fortune for others, and Drama for yet a third set of circumstances, and the game would show how the three circumstances were thus made dramtically different).
\n* Demonstrates the differences between at-the-Begininng/in-the-Middle/at-the-End types of resolution.
\n
\nAnd so on.
\n
\nGuidelines
\n
\n* Try and make the link between the theory and your game as explicit as possible without obfuscating the game itself. Remember one of the aims is to demonstrate something in action. Perhaps marginal notes would be good for this, like in The Mountain Witch!
\n
\n* Try and mix the theory into the game in a cool way, with symbolic/thematic elements mixing between the two. Your game's whole world can be a metaphor for theory, or you can reinforce the theory-elements with suitably symbolic analogues.
\n
\n* Don't spend too much time on it! Keep it fun, simple, and frothy.

    \n This is a really neat idea/challenge. And in a sort of zeitgheist way it gels with some thinking I had been toying with over the weekend. I had been considering that the various elements I like (or want to like) about comic book/super hero gaming did not necessarially work together mechanically.

    For instance, from time to time I like really crunchy superheroic battles (Champions maybe?) but I also like the almost soap operatic back stories (With Great Power maybe?). I've yet to see a system handle both well. And I've a thing for legacy heroes (see the current JSA comics) and alternate realities (Elseworlds, What If?), but have yet to see a system that can provide mechanical linkages for such things.

    So what? So I was thinking, "Self, instead of one game to do them all, why not FOUR games that can interact but that each do ONE thing really well." Four being a fairly arbitrary number. Now I see this contest. So now I have to see if I can find some theory linkages that parrallel this design intention.

    Don't know if it will happen, but I did what to get that off my chest and say again, "Great Idea for a Challenge!"
