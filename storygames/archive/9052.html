    \n The PDF conversion process downsamples images by default unless you tell it not to. Compared with high-resolution images suitable for working with in Photoshop or whatnot, the resulting optimized-for-print images are rather tiny. Since your document is black-and-white, the space saving can be very large indeed. At 400kb - 500kb per full-page greyscale image, I'd say you're still likely at a decent resolution for print.
    \n
    \nIf I were you, I'd just check to make sure you didn't accidentally select optimize for web or screen viewing when the PDF was created. If not, I don't see a problem.

    \n What they said. If you saved the file as a PDF for use with Acrobat 9 rather than an earlier version, it would probably compress the file even more.

    BTW, I saw the SPITE preview-preorder PDF: It looks like you may have used the feature in Acrobat, Document-->Reduce File Size? This actually shrinks things a great deal, but it also makes the art look like crap up-close.

    Instead, go to the Advanced menu, and select "PDF Optimizer".

    To compare: Maid RPG has about a gig of art IIRC, non-compressed.
    When I prepared the "print version", with really high quality PDF exporting (to Adobe 8 format: Less than that, the file was a lot larger) it was about 40mb.
    When I prepared the "PDF version", I was able to get the file size down to 8mb with Reduce File Size, but the art looked like crap up close. When I prepared the PDF version with Acrobat's "PDF Optimizer" instead, it shrunk it down to about 12mb and I could not see any real difference in the quality of the art from the original 40mb version.

    Hope that helps. In Acrobat Pro: PDF Optimizer Rulez, Reduce File Size Drools, and save your PDF to be compatible with Acrobat readers 8 or higher for max file compression.

    -Andy

    \n Damn, the first time in five years that somebody in this subculture is using the same exact workflow I use, and I miss the question. Now I'll have to wait until something like 2014 for a chance to discourse on the fine points of pdf creation in Quark. Except that the whole program will probably be long obsolete at that point.

    Actually, looking at what the others wrote, I still have to add something: what you need to do is to look at the pdf creation settings of Acrobat systematically. It's not enough to just "create a pdf" with Acrobat, you need to make sure the settings are right. In fact, you should be using Acrobat Distiller (the pdf-creation program you get alongside Acrobat) and looking at the image sampling settings you're using: 300 dpi is an appropriate resolution for color and greyscale images for this use (1200 for bw images), so make sure your creation settings in that regard are correct. Also, better not to set the images to be packaged as jpg, as it might lose some detail for you.

    As Andy said, don't use the internal Acrobat function for smaller file sizes. It downscales images to something like 150 dpi and thus makes them unusable for most printing purposes. In general, don't mess with your print pdfs unnecessarily with anything where you don't know exactly what the program is doing - the file is the size it is, you won't have to deal with it after you get it to the printer. Burn it into a cd and send it in mail if you can't get it to its destination any other way. Better to have too much data than too little in your pdf file.

    The file size of 45 megabytes seems roughly appropriate for this sort of publication, so you probably have roughly the correct settings. But you can't know without looking, better to learn all the settings you can influence when making a pdf so you can make informed choices.

    Also, if the print proof looks fine to your eye, then you're essentially making a mountain out of a molehill. The proof looks just like the stuff they'll be sending to your customer, so if you're happy with it, why would anybody else care to complain?

    \n Images are at 300dpi already, so no downsampling would occur. Nonetheless, you can use ZIP Compression of all artwork for lossless reduction of file size. I always use it for print PDFs... for any PDFs, come to think of it, unless it's all vector art (i.e. almost no benefit from ZIPping).

    Font embedding can bloat it, too: subset fonts below 100% (i.e. it will subset every font, as you don't use every glyph in any font). The only reason one would want to subset fonts below 0% (i.e. embed them all) is if one is wanting a PDF recipient to be able to use the Touch Up text tool to make content changes. Not necessary (or even an option) with Lulu. But, yeah, if you aren't subsetting Ariel or Times, it will be about 300kb to embed it.

    But in the end, if Lulu doesn't bitch about the storage space or transfer time or (maybe) the printer lag, then fuck it: do as Eero says and Go Large!

    \n It is typical, with images to go from a large file to a smaller size when you create a PDF.

    The downsampling reduces the file size, but the printer still has to rip the file, thus taking a long time to actually turn your PDF into a printed document. Basically, the faster you convert the file to a PDF, the longer it's going to take the printer to rip that file and pull the element out that need to be printed. It's more complicated than that, but I'm just trying to paraphrase.

    650mb to 45mb is a big change, but it all depends on how much background art you have on a page. That's where the real ripping time comes in.

    Do you have a sample page you can show?

    EDIT: Also note that Quark to PDF is not an exact science at the moment. Adobe still owns the PDF technology and they like that INDESIGN and ACROBAT talk well with each other and other programs don't. I've had lot of little weird things happen from QUARK to PDF.

    \n I'm going to assume you can't just change the file properties to the 6x9 size in Word, because you don't want the book to reflow, right? Because if you haven't done a lot of hand-positioning of content or manual flow-control, just resize the page in Word and print to PDF.

    So, if that's no good, try this: Most printer drivers I've seen have an option, under a Paper Handling area, to set Page Scaling to Fit to Printable Area. So set the "Paper" for the printer (click the Properties button on the Print dialog box) to be the size you are aiming for, set Fit to Printable Area and Auto-Rotate and Center, and then Print to File (a .PS file). Then PS -> PDF with Distiller or similar.

    Should work. Haven't tested. Note that the aspect ratios of 6x9 is different from 8.5x11, so you'll get whitespace on two of the page edges (top and bottom, I think--math dyslexic).
